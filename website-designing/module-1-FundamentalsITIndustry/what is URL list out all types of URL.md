# What is URL, List out all types of URLs
A URL (Uniform Resource Locator) is a specific type of URI (Uniform Resource Identifier). It is the unique character string used as a web address to locate and access specific resources (like web pages, images, or files) on the internet.
URLs can be broken down into various categories based on their structure (how they reference a file) and their purpose (how they function for users and search engines).

1. Structure-Based URLs
Absolute URL:
Contains the complete address needed to locate a resource, including the protocol (e.g., https://), domain name, and full file path.
'''
https://google.com
'''

Relative URL: 
Contains only the path to the file relative to the current webpage's base directory. It omits the protocol and domain.
'''
/search/about.html or ../images/logo.png
'''

Protocol-Relative URL:
Omits the protocol (http:// or https://) at the beginning, allowing the browser to automatically use whatever protocol the current page is using.
'''
//://googleapis.com
'''

2. SEO & Web Design URLs
Canonical URL:
A specific HTML link element (rel="canonical") used by search engines to designate the "master" or preferred version of a webpage to prevent duplicate content issues.

Static URL:
A fixed, unchanging URL that directly points to a specific file on a server. They are generally simple, do not have complex parameters, and are highly SEO-friendly.

Dynamic URL:
A URL generated by a database or server script based on user actions, inputs, or database queries. They typically contain parameters like question marks, equal signs, and ampersands.

Friendly / Clean URL:
A URL structured without strange characters or long strings of numbers, designed to be easily readable and understandable by human users.

3. Protocol / Scheme-Based URLs (Access Mechanisms)
The prefix of a URL dictates how the web browser or client should retrieve the resource:
HTTP/HTTPS:
Hypertext Transfer Protocol/Secure. The standard URLs for accessing secure/non-secure web pages.

FTP:
File Transfer Protocol. Used for transferring files between computers on a network.

Mailto:
Used to open the default email client and create a new email draft addressed to the specified email address.

File:
Used to access files stored locally on the user's own computer or device.

Telnet:
Used for remote host connectivity and terminal sessions.

4. Special Purpose URLs
Short URL:
A compressed version of a long, clunky URL used primarily for sharing in character-limited spaces (like social media platforms).
'''
https://bit.ly
'''

Vanity / Branded URL:
A custom short URL used to build brand awareness rather than just hiding a long link.
'''
https://nyti.ms
'''

Deep Link:
A URL that directs the user to a specific in-app location or deeper page within a website's hierarchy, rather than just the homepage.
